# Security Policy

## Supported Versions

PoWV Protocol is currently in pre-release development.

At this stage, there are no public production releases or versioned software distributions.

Security reviews are conducted internally as part of ongoing architecture and protocol development.

---

## Reporting a Vulnerability

If you believe you have identified a security vulnerability related to PoWV Protocol, please report it responsibly.

Please include:

* Description of the issue
* Potential impact
* Reproduction steps (if applicable)
* Proof of concept (if available)

### Contact

Security reports should be sent to:

powv.protocol@proton.me

(Temporary contact may be provided through official channels until dedicated infrastructure is established.)

---

## Disclosure Policy

Please do **not disclose vulnerabilities publicly** before coordinated review.

PoWV Protocol follows responsible disclosure principles.

Valid reports will be reviewed and assessed based on:

* Severity
* Exploitability
* Protocol impact
* Infrastructure risk

We aim to acknowledge reports within a reasonable timeframe.

---

## Scope

Security considerations include, but are not limited to:

* Protocol architecture
* Cryptographic mechanisms
* Consensus logic
* Smart contract vulnerabilities
* Oracle attack vectors
* Infrastructure trust boundaries

---

Thank you for helping improve PoWV security.
